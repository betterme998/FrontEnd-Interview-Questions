# 邂逅前端就业指导

## 1.课程核心内容

.不仅仅是针对前端求职，其他岗位也是一样  
.不仅是简单的给你一份简历模板  
.不只是简单的告诉你怎么投递简历，怎么做面试前的准备，怎么面试时表现的更好  
.不只是某一两个知识点，不只是我给你讲的闭包，Vue原理/源码  
.也不是我告诉你几种性能优化的方式和面试怎么回答

.授人以鱼，不如授人以渔

.比如我们讲简历：  
.如果我只是告诉你一套简历模板你照着抄就行了，那么你就只会套简历模板  
.但是如果我给你讲的是一整套编写简历的思路，方法，那么你才能因地适宜，根据自己不同的实际情况，妥善的修改自己的简历  
.根据面试岗位的不同，合理的去修改和投递简历

.比如我们要投递vue2/vue3，这个时候就不要过多的去突出小程序，react，node Nginx等能力，招的是vue2，vue3的岗位，你就应该突出你的vue2/3的能力，写项目就因该写vue2/vue3的项目，比如编写过Vue3的UI组件库，，突出这样的能力，投递岗位时才能更加的匹配，更加匹配看到简历才能优先选择你的简历，但如果你使用通用简历，简历有很多页，但是前面不断在强调小程序，react有多厉害，人家又不要小程序，不要react，所以我们讲简历时讲的是一整套编写简历的思路，而不是现在直接写一份简历就可以了，我们讲简历时，尽力会讲项目经验，很多人不会讲项目经验，我不知道我项目的亮点啊，也不知道项目的难点啊，我不会写啊。有些同学怎么写项目经验：1.使用vue3开发后台管理系统.2.使用Vue-Router开发路由功能（动态路由）.3.Vuex/Pinia管理状态（统一管理） 。如果是这样写的话，那就是千篇一律，现在很多人写简历都是这样写的，如果你说我还有更多，确实有更多但是说了等于没说，对于很多同学4.对axios二次封装，进行网络请求，但是你措辞可能会好一点
