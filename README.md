## 1.课程核心内容

.不仅仅是针对前端求职，其他岗位也是一样  
.不仅是简单的给你一份简历模板  
.不只是简单的告诉你怎么投递简历，怎么做面试前的准备，怎么面试时表现的更好  
.不只是某一两个知识点，不只是我给你讲的闭包，Vue原理/源码  
.也不是我告诉你几种性能优化的方式和面试怎么回答

.授人以鱼，不如授人以渔

.比如我们讲简历：  
.如果我只是告诉你一套简历模板你照着抄就行了，那么你就只会套简历模板  
.但是如果我给你讲的是一整套编写简历的思路，方法，那么你才能因地适宜，根据自己不同的实际情况，妥善的修改自己的简历  
.根据面试岗位的不同，合理的去修改和投递简历

.比如我们要投递vue2/vue3，这个时候就不要过多的去突出小程序，react，node Nginx等能力，招的是vue2，vue3的岗位，你就应该突出你的vue2/3的能力，写项目就因该写vue2/vue3的项目，比如编写过Vue3的UI组件库，，突出这样的能力，投递岗位时才能更加的匹配，更加匹配看到简历才能优先选择你的简历，但如果你使用通用简历，简历有很多页，但是前面不断在强调小程序，react有多厉害，人家又不要小程序，不要react，所以我们讲简历时讲的是一整套编写简历的思路，而不是现在直接写一份简历就可以了，我们讲简历时，尽力会讲项目经验，很多人不会讲项目经验，我不知道我项目的亮点啊，也不知道项目的难点啊，我不会写啊。有些同学怎么写项目经验：1.使用vue3开发后台管理系统.2.使用Vue-Router开发路由功能（动态路由）.3.Vuex/Pinia管理状态（统一管理） 。如果是这样写的话，那就是千篇一律，现在很多人写简历都是这样写的，如果你说我还有更多，确实有更多但是说了等于没说，对于很多同学4.对axios二次封装，进行网络请求，但是你措辞可能会好一点.后面可能写了很多条，但是一看每个人写这个东西千篇一律，没有任何区别，如果这个东西出现在4，5年前的时候，说句不夸张的话，你只要会去用vue开发组件，找工作就没有问题了，很多年前这样写简历的时候没有问题，那个时候写这个东西会发现你会的东西还挺多的，最早讲课的时候对这个axios二次封装，现在很多人因该都做过二次封装，这个在网上也有很多的整理，到底要二次封装还是不要二次封装，我在讲直播课的时候提到过，这个是因为我以前再做ios开发的时候，遇到过对应的一个问题，所以呢对所以的网络请求看我都会进行二次封装的，现在大家都二次封装都不知道原因，总之以前这样写简历是没有问题的，但是现在再去这样写简历，千篇一律。简历一定会被pass，一定不能去这样写项目经验，那么要怎么去写项目经验呢，到时候我会给大家一整套不管是项目经验，工作经验，相关的一下思路和方法，到时候如果会编写并且你已经完全掌握了这样的思路，把我给你的项目简历模板内容都可以吃透的情况下，你再把自己的简历，根据自己的情况做一些修改，妥善的去修改自己的简历，再根据不同的岗位需求，然后因地适宜去编写一个更加匹配的简历，那么你会发现你得到的面试机会，大大增加，绝对比以前大的多的，在找工作的时候最重要的并不是你的技术有多好，而是越匹配越好的，你和对应的岗位匹配才是最重要的，投递简历也是这样的，面试时回答问题也是这样的，我希望大家掌握的是这样的思路和方法，而不只是看到某一个点，这个是我们课程给大家讲的一共东西，比如我们有一个简历，这边有一共岗位招聘react，我们不是因该不断强调自己react能力吗。当然后面的项目可能有react，vue，低代码，BFF，小程序，最重要的是要掌握编写简历里面，这些项目经验也好，工作经验也好，它对应的思路和方法，而不只是会照抄而已，这个是我希望大家在课堂去掌握的东西，我想要做到的是授人以鱼不如授人以渔，我接下来要讲的东西就是关于面试八股文，

.在讲面试八股文时也是非常重要的，再学习八股文时很多同学单纯的是去背八股文，但是事实上可以做得更多，做的更好的。  
怎么样做得更多，做得更好呢？

.所有的题目都至少3层解答：  
1.第一层：概念+定义， 八股文

2.第二层：你自己的理解和见解，在学习中对它的思考 （闭包到底有什么用，没有闭包的话会产生什么后果，这个在以前课程有讲到过的）（闭包是为了引用外层作用域相关变量内容，但是如果没有闭包的话，这些都行都需要传进去的，传进去就会有很多参数，非常的麻烦，可以加上自己的思考，而不是去背八股文）

3.第三层：在平时工作中应用，遇到的问题，解决的方案（可以在面试的时候回答一下，在工作当中遇到那些闭包的问题，比如之前闭包怎么产生的内存泄露，有一些先关的对象消除不了，产生了内存泄露，然后我是怎么解决的，然后通过如何解决闭包来优化项目性能的，这些也可以回答。我们的技术点再项目当中是如何应用的）

4.第四层：通过面试官问你的的八股文，你刻意去引导到自己熟悉的领域  
比如问道你闭包，你引导到React中目前流行使用Hooks开发，但是Hooks中使用闭包内存泄漏问题。然后你就告诉他之前遇到的这个问题，这样就不仅仅在回答闭包的问题了，你还把对于交流的题目闭包，引申到了React，甚至引申到了Hooks，但是前提是你对React和Hooks都非常的懂，在你非常懂的前提下，面试官会发现我们再招react人才，这个人会react，我问你下他react题目吧，这个时候引导到这里的时候，因为有提前做好准备，就可以和面试官侃侃而谈，你会发现你们的面试过程就不再是一问一答了，再面试的过程中非常忌讳一问一答，像审犯人一样，问你一个问题你回答，问你一个问题你回答，其实这样的交流和沟通方式是不好的，甚至有点尴尬的，你们并没有真正再交流技术，面试的过程是交流技术的过程，以前提到的：候机法则，面试的过程要符合候机法则的，你和面试官通用再等待飞机起飞的时候，你们愿不愿意一起交流一下技术，如果能达到这样的效果是最好的，（后面会讲），如果你能把某一个面试题慢慢慢慢引导到你收悉的领域，准备好的领域，你们就不再是一问一答的，是真正交流技术，而且你可以侃侃而谈，聊到收悉的领域就可以发挥自我，说到很多很多你对react的理解，对hooks的理解，如果会react源码，可以聊聊react里面的对象，这个时候面试上的概率就非常高

比如问道你ES6之后的新特性，你引导Proxy，Reflect进一步引导到Vue3源码中响应式原理的实现。从object.defineProperty改成了Proxy,Reflect,为什么要改，源码里面是怎么做的，这样做有什么好处，你又从ES6引导到Vue3源码，又是你收悉的领域，你说我对Vue3源码不熟悉啊，不熟悉没有关系，学好我们的课程就可以了，我们的课程会和大家讲到的，到时候我会带领你阅读源码，并且·把那些Vue3常见的原理面试题，源码面试题，全部一个一个进行剖析的，所以不用担心，我希望的是大家在学习这门课程的时候，不仅仅是学会一道面试题就可以，两道面试题就可以了，你要学到的是面试题回答的方式，怎么去可以的引导面试官，这样的话才和面试官沟通的非常的顺畅，整个沟通的过程而不是很累，甚至你们面试完之后还会有一种意犹未尽的感觉，面试官有一种感觉就是Yes i do ,就是这个人了，感觉和hr说，我现在就要这个人，要给他这样的感觉，因为你们还没有聊完呢，还想聊更多呢，这是一个沟通交流的一个过程，是一个技术分析的过程，而不是一问一答，这种感觉是非常不好的，当然我还说了每个面试官都是不一样，有些面试官可能你回答第一个层次就行了，因为他可能对后面的知识也不懂，有些面试官可能技术不怎么样你回答对了可能还说你错了。他可能在网上看了一两篇文章，也不知道文章的来源，就草草的说错了，可能是那篇文章错了。这个时候不要指正他，不要说你错了，我看过源码怎么怎么样，指正他感觉就不好了。只要你碰到的是真正喜欢技术的面试官，你聊的越多越深，他就越感兴趣。你们就会聊的越投机越投缘，就想要和你以后共同工作，共同开发，共同一起做技术，我遇到这样的人就很愿意和他交朋友，招他一起进来。但一问一答那种模式非常非常差。最重要的一点是，你将面试引入到了自己收悉的领域。
